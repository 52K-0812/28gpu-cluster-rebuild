# 🛡️ [Part 2] 쿠버네티스 인증서 갱신 및 노드 동기화

## 1. 개요

| 항목          | 내용                                                                            |
| ------------- | ------------------------------------------------------------------------------- |
| **목적**      | 5년간 중단되어 인증서가 만료된 K8s 클러스터 정상화 및 GPU 워커 노드 재연결      |
| **대상**      | Master 노드, DGX Station, 2080Ti 워커 노드 3대                                  |
| **핵심 전략** | 만료된 인증서 갱신 후 SCP 우회 복사(Staging) 방식으로 워커 노드에 안전하게 배포 |

---

## 2. 문제 현상

- `kubectl get nodes` 실행 시 `localhost:8080` 연결 거부 에러
- 워커 노드 전체 `NotReady` 상태, 익명 사용자(`anonymous`)로 간주되어 마스터 연결 거부
- Kibana, Kafka 등 서비스 포드 다수 `CrashLoopBackOff`

---

## 3. 원인 분석

| 문제               | 원인                                                           |
| ------------------ | -------------------------------------------------------------- |
| kubelet 미기동     | 스왑(Swap) 메모리 활성화 상태 (K8s는 스왑 비활성화 필수)       |
| API 서버 연결 거부 | 2021년 이후 방치로 K8s 내부 인증서(유효기간 1년) 전량 만료     |
| 워커 노드 NotReady | 워커 노드가 구형(2021년) 인증서를 보유, 마스터와 인증 불일치   |
| SCP 직접 복사 실패 | `/etc/kubernetes/` 는 root 소유 보안 영역, 일반 유저 접근 거부 |

---

## 4. 해결 과정

### 4.1 마스터 노드 — kubelet 장애 및 인증서 갱신

스왑 비활성화 후 갱신된 `admin.conf`를 `kubelet.conf`에 주입하여 재가동.

**🛠️ 사용 명령어:**

```bash
sudo systemctl status kubelet                     # 엔진 상태 조회
sudo kubeadm alpha certs check-expiration         # 인증서 유효기간 진단 (EXPIRED 확인)
sudo swapoff -a                                   # 스왑 메모리 강제 비활성화
sudo cp /etc/kubernetes/admin.conf /etc/kubernetes/kubelet.conf  # 최신 인증서 주입
sudo systemctl restart kubelet                    # 엔진 재시작
sudo systemctl restart docker                     # 컨테이너 네트워크 초기화
```

### 4.2 kubectl 유저 권한 및 환경 변수 복구

`.kube` 디렉토리 소유권이 `root`에 묶여 있어 일반 유저로 `kubectl` 실행 불가. 소유권 일괄 변경으로 해결.

**🛠️ 사용 명령어:**

```bash
mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown -R $USER:$USER $HOME/.kube   # 소유권 일괄 변경 (핵심)
sudo chmod -R 755 $HOME/.kube
unset KUBECONFIG                        # 꼬인 환경변수 삭제
kubectl get nodes                       # sudo 없이 실행 확인
```

---

### 4.3 워커 노드 — 인증서 동기화 (SCP 우회 복사)

보안 영역(`/etc/kubernetes/`)에서 직접 SCP 복사 시 권한 오류 발생. 일반 유저 영역으로 파일을 먼저 꺼낸 뒤(Staging) 워커 노드에서 당겨오는 방식으로 우회.

**🛠️ 사용 명령어:**

```bash
# [마스터 노드] 일반 유저 영역으로 파일 이동 후 소유권 해제
sudo cp /etc/kubernetes/admin.conf ~/for_worker.conf
sudo chown ubuntu:ubuntu ~/for_worker.conf

# [워커 노드] 새 인증서 획득 및 재기동
sudo swapoff -a
sudo mv /etc/kubernetes/kubelet.conf /etc/kubernetes/kubelet.conf.old  # 기존 파일 백업
scp ubuntu@(control-plane-public-ip):~/for_worker.conf ~/new_admin.conf            # 마스터에서 당겨오기
sudo mv ~/new_admin.conf /etc/kubernetes/kubelet.conf
sudo systemctl restart kubelet

# [마스터 노드] 최종 상태 확인
kubectl get nodes                       # 전체 노드 Ready 확인
```
