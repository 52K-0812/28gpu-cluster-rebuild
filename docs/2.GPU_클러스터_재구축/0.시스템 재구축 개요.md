# GPU 클러스터 재구축 개요

> **작성 일자:** 2026-03-20 **목적:** 재구축 전 전체 설계 구조 및 작업 계획 정리

---

## 1. 클러스터 구성 설계

| 역할          | 하드웨어       | 호스트네임    | IP                        | 주요 역할          |
| ------------- | -------------- | ------------- | ------------------------- | ------------------ |
| Control Plane | HPE ProLiant   | master-01     | (control-plane-public-ip) | K8s 제어 평면      |
| Worker        | HPE ProLiant   | master-02     | (control-plane-public-ip) | 일반 워크로드      |
| GPU Worker    | NVIDIA DGX     | v100-gpu-01   | (control-plane-public-ip) | V100 × 4           |
| GPU Worker    | Supermicro     | 2080ti-gpu-02 | (control-plane-public-ip) | 2080Ti × 8         |
| GPU Worker    | Supermicro     | 2080ti-gpu-03 | (control-plane-public-ip) | 2080Ti × 8         |
| GPU Worker    | Supermicro     | 2080ti-gpu-04 | (control-plane-public-ip) | 2080Ti × 8         |
| Storage       | Supermicro NAS | nas-01        | (control-plane-public-ip) | 28TB 공유 스토리지 |

> **네이밍 정책 변경:** 기존 레거시 시스템의 혼재된 네이밍(cheetah, v100dgx 등)을 `역할-번호` 체계로 통일했습니다. 운영 중 역할 파악이 즉시 가능하도록 설계했습니다.

---

## 2. 소프트웨어 스택 설계

| 단계              | 구성 요소            | 선택 이유                            |
| ----------------- | -------------------- | ------------------------------------ |
| OS                | Ubuntu 22.04 LTS     | K8s 및 NVIDIA 드라이버 공식 지원     |
| Container Runtime | Containerd           | K8s 1.24+ 표준 런타임                |
| Orchestration     | Kubernetes v1.29     | 안정적인 LTS 수준 버전               |
| CNI               | Calico v3.27         | 프로덕션 검증, NetworkPolicy 지원    |
| GPU 관리          | NVIDIA GPU Operator  | DGX + Supermicro 혼합 환경 자동 관리 |
| 스토리지          | NFS Provisioner      | NAS 28TB를 K8s PV로 연결             |
| 모니터링          | Prometheus + Grafana | GPU 온도/전력/메모리 실시간 시각화   |
| 관리 UI           | Portainer            | 웹 기반 클러스터 관리                |
| 원격 접속         | Tailscale            | 외부에서 안전한 VPN 접속             |

---

## 3. 네트워크 설계

```
[외부 접속]
    │
    └── Tailscale VPN (100.76.84.68)
    └── MetalLB LoadBalancer ((control-plane-public-ip))
         │
[1G 관리망 - (control-plane-public-ip)]
    ├── master-01 (151)  ← K8s API, 관리 트래픽
    ├── master-02 (152)
    ├── v100-gpu-01 (153)
    ├── 2080ti-gpu-02 (154)
    ├── 2080ti-gpu-03 (155)
    ├── 2080ti-gpu-04 (156)
    └── nas-01 (157)
         │
[10G 데이터망 - 10.10.10.x]
    ├── v100-gpu-01 ((storage-backend-ip))
    ├── 2080ti-gpu-02 ((storage-backend-ip))
    ├── 2080ti-gpu-03 ((storage-backend-ip))
    ├── 2080ti-gpu-04 ((storage-backend-ip))
    └── nas-01 ((storage-backend-ip))  ← 학습 데이터 전송 전용
```

> **망 분리 이유:** K8s 관리 트래픽(1G)과 AI 학습 데이터 전송(10G)을 분리해 관리 통신이 대용량 데이터 전송에 영향을 받지 않도록 설계했습니다.

---

## 4. 작업 순서

```
1. 데이터 백업 (레거시 → NAS)
2. Ubuntu 22.04 클린 설치 (전 노드)
3. K8s 클러스터 초기화 (master-01)
4. Calico CNI 설치
5. 워커 노드 Join (master-02 먼저 검증 후 나머지)
6. 10G NIC 이전 (하드웨어 마지막 작업)
7. GPU Operator 설치
8. NFS Provisioner 설치
9. Prometheus + Grafana 설치
10. MetalLB + Tailscale 설치
```
