# 레거시 시스템 데이터 백업 및 마이그레이션

> **작업 일자:** 2026-03-20
> **대상 장비:** CHEETAH 레거시 서버 전체
> **목적:** 전면 재구축 전 기존 데이터 안전하게 보존

---

## 1. 작업 개요

Ubuntu 22.04 클린 설치 전에 기존 CHEETAH 시스템의 사용자 데이터와 설정 파일을 NAS로 백업했습니다. 재구축 후 필요한 데이터를 복원할 수 있도록 데이터 연속성을 확보하는 것이 목적입니다.

| 항목          | 내용                                                  |
| ------------- | ----------------------------------------------------- |
| **백업 대상** | `/home/ubuntu` (홈 디렉토리 전체)                     |
| **저장 위치** | NAS (`(control-plane-public-ip):/volume1/nas_backup`) |
| **압축 방식** | tar.gz (Gzip)                                         |

---

## 2. 백업 방식 선택 이유

### 단순 복사 대신 tar 아카이빙을 선택한 이유

수만 개의 파일을 하나씩 복사하면 두 가지 문제가 생깁니다.

1. **속도 저하** — 파일 수가 많을수록 I/O 오버헤드가 누적됩니다.
2. **권한 정보 손실** — NFS 마운트 환경에서 단순 복사 시 Linux 파일 권한(`chmod`)과 심볼릭 링크가 깨질 수 있습니다.

`tar -czvf`는 파일 권한, 소유자, 심볼릭 링크를 그대로 보존하면서 단일 파일로 압축합니다.

---

## 3. 백업 실행

### NAS 마운트

```bash
# 마운트 지점 생성
sudo mkdir -p /mnt/nas_backup

# NFS 마운트
sudo mount -t nfs (control-plane-public-ip):/volume1/nas_backup /mnt/nas_backup
```

### 데이터 압축 백업

```bash
# 홈 디렉토리 전체 압축 백업
sudo tar -czvf /mnt/nas_backup/ubuntu_home_backup.tar.gz /home/ubuntu
```

### 백업 파일 무결성 확인

```bash
# 파일 생성 여부 및 용량 확인
ls -lh /mnt/nas_backup/ubuntu_home_backup.tar.gz
```
