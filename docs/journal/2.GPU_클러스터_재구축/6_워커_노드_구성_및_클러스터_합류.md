# 워커 노드 구성 및 클러스터 합류

> **작업 일자:** 2026-03-20 ~ 21
> **대상 장비:** master-02, v100-gpu-01, 2080ti-gpu-02~04 (총 5대)
> **목적:** 마스터 노드 Taint 해제 후 워커 노드 6대 클러스터 합류

---

## 1. 작업 개요

마스터 노드 Taint를 해제하고, 워커 노드들을 클러스터에 합류시킵니다. 6대를 한번에 진행하기 전에 **master-02로 먼저 검증**한 뒤 나머지를 순차적으로 진행했습니다.

---

## 2. 마스터 노드 Taint 해제

Kubernetes는 기본적으로 Control Plane 노드에 일반 Pod를 배치하지 않습니다. 이 클러스터는 학습/연구 목적이므로 마스터 노드도 워크로드를 처리할 수 있도록 Taint를 해제했습니다.

```bash
kubectl taint nodes --all node-role.kubernetes.io/control-plane-
```

> **명령어 끝의 `-` 의미:** Taint를 추가하는 게 아니라 **제거**하는 명령입니다.

---

## 3. 워커 노드 환경 구성 (원샷 스크립트)

각 워커 노드 터미널에서 아래 스크립트를 통째로 붙여넣으면 됩니다.

```bash
# 1. Swap 비활성화 및 커널 모듈 설정
sudo swapoff -a && sudo sed -i '/ swap / s/^\(.*\)$/#\1/g' /etc/fstab
cat <<EOF | sudo tee /etc/modules-load.d/k8s.conf
overlay
br_netfilter
EOF
sudo modprobe overlay && sudo modprobe br_netfilter

# 2. 커널 파라미터 적용
cat <<EOF | sudo tee /etc/sysctl.d/k8s.conf
net.bridge.bridge-nf-call-iptables  = 1
net.bridge.bridge-nf-call-ip6tables = 1
net.ipv4.ip_forward                 = 1
EOF
sudo sysctl --system

# 3. Containerd 설치 및 설정
sudo apt-get update && sudo apt-get install -y containerd
sudo mkdir -p /etc/containerd
containerd config default | sudo tee /etc/containerd/config.toml
sudo sed -i 's/SystemdCgroup = false/SystemdCgroup = true/g' /etc/containerd/config.toml
sudo systemctl restart containerd

# 4. Kubernetes 도구 설치 (v1.29)
curl -fsSL https://pkgs.k8s.io/core:/stable:/v1.29/deb/Release.key | \
  sudo gpg --dearmor -o /etc/apt/keyrings/kubernetes-apt-keyring.gpg
echo 'deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring.gpg] \
  https://pkgs.k8s.io/core:/stable:/v1.29/deb/ /' | \
  sudo tee /etc/apt/sources.list.d/kubernetes.list
sudo apt-get update
sudo apt-get install -y kubelet kubeadm kubectl
sudo apt-mark hold kubelet kubeadm kubectl
```

## 4. master-02 먼저 Join 검증

워커 노드 5대를 한번에 붙이기 전에 master-02로 먼저 검증했습니다.

```bash
# master-01에서 토큰 발급
kubeadm token create --print-join-command

# master-02에서 실행
sudo kubeadm join MASTER_IP:6443 --token [토큰] \
  --discovery-token-ca-cert-hash sha256:[해시값]

# master-01에서 상태 확인
kubectl get nodes
```

> **먼저 검증한 이유:** 6대를 한번에 Join하다가 토큰 문제나 설정 오류가 있으면 원인 파악이 어렵습니다. 한 대로 먼저 검증하고 나머지를 진행하는 것이 안전합니다.

## 5. 나머지 워커 노드 Join (153~156)

master-02 검증 완료 후 나머지 4대를 순차적으로 Join했습니다.

```bash
sudo kubeadm join MASTER_IP:6443 --token [토큰] \
  --discovery-token-ca-cert-hash sha256:[해시값]
```

> 토큰 만료(24시간) 시 master-01에서 `kubeadm token create --print-join-command`로 재발급.

## 6. NFS 스토리지 연결

모든 워커 노드에서 NAS(28TB)에 접근할 수 있도록 NFS 클라이언트를 설정합니다.

```bash
# NFS 도구 설치
sudo apt install -y nfs-common

# 마운트 폴더 생성 및 연결
sudo mkdir -p /data
sudo mount -t nfs NAS_1G_IP:/data /data

# 재부팅 후 자동 마운트 설정
echo "NAS_1G_IP:/data /data nfs defaults 0 0" | sudo tee -a /etc/fstab

# 연결 확인
df -h | grep /data
```
