# Kubernetes 클러스터 초기화 (Master Node)

> **작업 일자:** 2026-03-20
> **대상 장비:** master-01 (MASTER_IP)
> **목적:** 독립 서버를 Kubernetes Control Plane으로 구성

---

## 1. 작업 개요

`kubeadm init` 명령어로 master-01을 클러스터의 제어 평면(Control Plane)으로 초기화합니다. 이 단계가 완료되면 워커 노드들이 Join할 수 있는 토큰이 발급됩니다.

---

## 2. 사전 환경 설정

### Swap 비활성화

```bash
sudo swapoff -a
sudo sed -i '/swap/s/^/#/' /etc/fstab
```

> **이유:** Kubernetes 스케줄러는 각 노드의 메모리를 정확히 계산해서 Pod를 배치합니다. Swap이 활성화되어 있으면 실제 메모리와 가용 메모리 계산이 틀어져 스케줄링 오류가 발생합니다. K8s 공식 문서에서도 Swap 비활성화를 필수로 요구합니다.

### 커널 모듈 및 IP 포워딩 활성화

```bash
sudo modprobe br_netfilter
echo 1 | sudo tee /proc/sys/net/ipv4/ip_forward
sudo sysctl --system
```

---

## 3. 클러스터 초기화

```bash
sudo kubeadm init \
  --pod-network-cidr=192.168.0.0/16 \
  --apiserver-advertise-address=MASTER_IP
```

| 옵션                            | 값                        | 이유                              |
| ------------------------------- | ------------------------- | --------------------------------- |
| `--pod-network-cidr`            | 192.168.0.0/16            | Calico CNI 권장 대역              |
| `--apiserver-advertise-address` | MASTER_IP | 워커 노드가 마스터를 찾을 실제 IP |

> **pod-network-cidr 이유:** 다음 단계에서 설치할 Calico CNI가 이 대역을 기준으로 Pod 간 가상 네트워크를 구성합니다. CNI와 대역이 맞지 않으면 Pod 간 통신이 불가합니다.

---

## 4. kubectl 권한 설정

초기화 직후 일반 사용자 계정에서 `kubectl`을 사용할 수 있도록 설정합니다.

```bash
mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config
```

> **이유:** `admin.conf`는 K8s API 서버 접근 인증서가 담긴 파일입니다. root 권한 없이 `kubectl`을 사용하려면 이 파일을 일반 유저 홈 디렉토리로 복사해야 합니다.

---

## 5. Join Token 발급

워커 노드들이 이 클러스터에 합류할 때 사용하는 토큰을 발급합니다.

```bash
kubeadm token create --print-join-command
```

**출력 예시:**

```text
kubeadm join MASTER_IP:6443 --token xxxxx \
  --discovery-token-ca-cert-hash sha256:xxxxx
```

> ⚠️ **토큰 유효기간 24시간.** 만료 시 위 명령어로 재발급하면 됩니다. 워커 노드 Join 작업 전 반드시 따로 저장해두어야 합니다.
