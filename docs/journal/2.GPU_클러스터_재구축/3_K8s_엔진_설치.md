# Kubernetes 엔진(Containerd) 및 도구 설치

> **작업 일자:** 2026-03-20
> **대상 장비:** master-01 (MASTER_IP)
> **목적:** Kubernetes 클러스터 구동을 위한 컨테이너 런타임 및 관리 도구 설치

---

## 1. 작업 개요

Kubernetes는 컨테이너를 실행할 런타임이 필요합니다. Docker 대신 **Containerd**를 선택한 이유는 Kubernetes 1.24 이후 공식 표준 런타임이며, 불필요한 레이어 없이 더 가볍고 안정적으로 동작하기 때문입니다.

| 항목                  | 선택       | 이유                                                             |
| --------------------- | ---------- | ---------------------------------------------------------------- |
| **Container Runtime** | Containerd | K8s 공식 표준, Docker보다 경량                                   |
| **K8s 버전**          | v1.29      | LTS 수준의 안정 버전,구축 당시 생태계 안정성을 고려해 v1.29 채택 |
| **Cgroup Driver**     | systemd    | Ubuntu 22.04 권장 방식                                           |

## 2. 시스템 사전 설정

Kubernetes 노드 간 통신을 위해 커널 모듈과 네트워크 브릿지를 활성화합니다.

```bash
# 커널 모듈 로드 설정
cat <<EOF | sudo tee /etc/modules-load.d/k8s.conf
overlay
br_netfilter
EOF

sudo modprobe overlay
sudo modprobe br_netfilter

# 네트워크 브릿지 트래픽 허용
cat <<EOF | sudo tee /etc/sysctl.d/k8s.conf
net.bridge.bridge-nf-call-iptables  = 1
net.bridge.bridge-nf-call-ip6tables = 1
net.ipv4.ip_forward                 = 1
EOF

sudo sysctl --system
```

> **왜 필요한가:** K8s는 Pod 간 통신에 브릿지 네트워크를 사용합니다. `br_netfilter` 모듈이 없으면 iptables가 브릿지 트래픽을 처리하지 못해 Pod 간 통신이 차단됩니다.

## 3. Containerd 설치 및 설정

```bash
# 설치
sudo apt update
sudo apt install -y containerd

# 기본 설정 파일 생성
sudo mkdir -p /etc/containerd
containerd config default | sudo tee /etc/containerd/config.toml > /dev/null

# SystemdCgroup 활성화 (중요)
sudo sed -i 's/SystemdCgroup = false/SystemdCgroup = true/g' /etc/containerd/config.toml

# 서비스 재시작 및 자동 실행 등록
sudo systemctl restart containerd
sudo systemctl enable containerd
```

> **SystemdCgroup = true 이유:** Ubuntu 22.04는 systemd로 cgroup을 관리합니다. Containerd와 K8s가 동일한 cgroup 드라이버를 사용해야 충돌이 없습니다. 이 설정을 빠뜨리면 kubelet이 정상 동작하지 않습니다.

## 4. Kubernetes 도구 설치 (v1.29)

```bash
# 보안 패키지 및 GPG 키 추가
sudo apt install -y apt-transport-https ca-certificates curl gpg
sudo mkdir -p -m 755 /etc/apt/keyrings
curl -fsSL https://pkgs.k8s.io/core:/stable:/v1.29/deb/Release.key | \
  sudo gpg --dearmor -o /etc/apt/keyrings/kubernetes-apt-keyring.gpg

# 저장소 등록
echo 'deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring.gpg] \
  https://pkgs.k8s.io/core:/stable:/v1.29/deb/ /' | \
  sudo tee /etc/apt/sources.list.d/kubernetes.list

# 설치 및 버전 고정
sudo apt update
sudo apt install -y kubelet kubeadm kubectl
sudo apt-mark hold kubelet kubeadm kubectl
```

> **버전 고정 이유:** `apt-mark hold`로 버전을 고정하지 않으면 `apt upgrade` 시 K8s 버전이 자동으로 올라가 클러스터 버전 불일치가 발생할 수 있습니다.
