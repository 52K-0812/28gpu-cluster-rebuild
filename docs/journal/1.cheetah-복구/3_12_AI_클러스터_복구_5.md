# 🛡️ [Part 5] MetalLB LoadBalancer 및 외부 네트워크 접속 이슈

---

## 1. 개요

| 항목          | 내용                                                                          |
| ------------- | ----------------------------------------------------------------------------- |
| **목적**      | External-IP를 통한 외부 접속 불가 문제 해결 및 LoadBalancer 정상화            |
| **대상**      | `metallb-system` 네임스페이스의 controller, speaker 포드                      |
| **핵심 전략** | NodePort 우회로 앱 레이어 문제 여부를 먼저 확인 후, 이미지 저장소 교체로 해결 |

---

## 2. 문제 현상

- LoadBalancer로 부여받은 External-IP(LB_PUBLIC_IP) 외부 접속 불가
- 외부에서 Ping 시 `Destination Host Unreachable`
- `metallb-system` 내 controller, speaker 포드 `ImagePullBackOff` 상태

---

## 3. 원인 분석

NodePort(LB_PUBLIC_IP:31969)로 접속 시 성공 → 애플리케이션 문제가 아닌 LoadBalancer 통로 문제로 확정.

MetalLB 포드 상세 조회 결과 `Failed to pull image "metallb/controller:v0.9.3"` 에러 발견. 5년 전 설정된 Docker Hub 이미지가 삭제되었거나, MetalLB 공식 저장소가 `quay.io`로 이전되어 이미지 Pull 자체가 불가능한 상태.

**🛠️ 사용 명령어 (진단):**

```bash
kubectl describe pod [Controller-Pod-Name] -n metallb-system
kubectl describe pod [Speaker-Pod-Name] -n metallb-system
# Failed to pull image "metallb/controller:v0.9.3" 확인
```

---

## 4. 해결 과정

### 4.1 Controller 이미지 저장소 변경

**🛠️ 사용 명령어:**

```bash
kubectl set image deployment/controller \
  controller=quay.io/metallb/controller:v0.9.3 \
  -n metallb-system
```

---

### 4.2 Speaker DaemonSet 이미지 저장소 변경

**🛠️ 사용 명령어:**

```bash
kubectl set image ds/speaker \
  speaker=quay.io/metallb/speaker:v0.9.3 \
  -n metallb-system
```

---

### 4.3 포드 상태 확인

**🛠️ 사용 명령어:**

```bash
kubectl get pods -n metallb-system -w
```

---

## 5. 결과

- MetalLB controller, speaker 포드 `1/1 Running` 상태로 완전 복구
- External-IP(LB_PUBLIC_IP) 라우팅 정상화
- 80 포트를 통한 외부 대시보드 직접 접속 환경 복구 완료
