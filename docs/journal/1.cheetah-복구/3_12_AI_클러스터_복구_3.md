# 🛡️ [Part 3] Elasticsearch & Kibana 메모리 이슈

---

## 1. 개요

| 항목          | 내용                                                              |
| ------------- | ----------------------------------------------------------------- |
| **목적**      | K8s 노드 복구 후 `CrashLoopBackOff` 상태의 Kibana 포드 정상화     |
| **대상**      | `cheetah-ecosystem` 네임스페이스의 Elasticsearch, Kibana          |
| **핵심 전략** | JVM Heap 증설 및 Replica 수 조정으로 단일 노드 환경에 맞게 재구성 |

---

## 2. 문제 현상

- `cheetah-kibana` 포드가 `CrashLoopBackOff` / `Error` 상태를 반복하며 재시작
- Kibana 대시보드 접속 불가

---

## 3. 원인 분석

로그에서 `circuit_breaking_exception`, `[parent] Data too large` 에러 확인.

Elasticsearch JVM Heap 메모리 할당량이 약 478MB로 설정되어 있었고, 5년치 누적 데이터를 로드하는 과정에서 OOM 방지를 위한 서킷 브레이커가 작동. 이로 인해 Kibana의 연결 요청이 강제 차단됨.

**🛠️ 사용 명령어 (진단):**

```bash
kubectl logs -n cheetah-ecosystem [Kibana-Pod-Name] --tail 50
kubectl get statefulset cheetah-es-cluster -n cheetah-ecosystem -o yaml | grep -A 5 resources
# 결과: resources: {} — 리소스 제한 없이 시스템 최솟값으로 동작 중
```

---

## 4. 해결 과정

### 4.1 Elasticsearch 컨테이너 메모리 한도 증설 (2Gi)

**🛠️ 사용 명령어:**

```bash
kubectl patch statefulset cheetah-es-cluster -n cheetah-ecosystem \
  --type='json' \
  -p='[{"op": "replace", "path": "/spec/template/spec/containers/0/resources", "value": {"limits": {"cpu": "1", "memory": "2Gi"}, "requests": {"cpu": "500m", "memory": "2Gi"}}}]'
```

---

### 4.2 JVM Heap 사이즈 증설 (1GB)

컨테이너 메모리 증설 후, Java 애플리케이션이 실제로 사용하는 Heap 크기를 `ES_JAVA_OPTS` 환경변수로 고정.

**🛠️ 사용 명령어:**

```bash
kubectl set env statefulset/cheetah-es-cluster \
  ES_JAVA_OPTS="-Xms1g -Xmx1g" \
  -n cheetah-ecosystem
```

---

### 4.3 Replica 수 조정 (Yellow → Green)

단일 노드 환경에서 Replica 생성 실패로 클러스터 상태가 `yellow`에 고정되는 문제. Replica 수를 0으로 강제 조정.

**🛠️ 사용 명령어:**

```bash
# Replica 0으로 변경
curl -X PUT "10.100.194.186:9200/_settings" \
  -H 'Content-Type: application/json' \
  -d '{"index":{"number_of_replicas":0}}'

# 클러스터 상태 확인 (green 확인)
curl -X GET "10.100.194.186:9200/_cluster/health?pretty"
```

---

### 4.4 Kibana 포드 재시작

정상화된 Elasticsearch와 재연결하기 위해 Kibana 포드 삭제 후 자동 재생성.

**🛠️ 사용 명령어:**

```bash
kubectl delete pod -n cheetah-ecosystem [Kibana-Pod-Name]
```

---

## 5. 결과

- Elasticsearch 클러스터 상태 `green` (active_shards_percent: 100.0%) 정상화
- Kibana 포드 `1/1 Running` 확인
