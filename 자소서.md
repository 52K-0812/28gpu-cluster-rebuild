# 자기소개서

---

## 1. 지원 동기

AI 인프라를 직접 설계하고 운영하는 엔지니어가 되고 싶습니다. 학과 서버실에서 5년간 방치된 GPU 클러스터를 발견하고, 담당 업체도 없는 상황에서 혼자 복구를 시작했습니다. GRUB 루트쉘 탈취부터 K8s 인증서 갱신, RAID 복구, 10G 네트워크 재구성까지 2주 만에 27GPU 클러스터를 운영 가능한 상태로 만들었습니다. 이 경험을 통해 AI 인프라가 실제로 어떻게 동작하는지를 몸으로 익혔고, 이 분야에서 전문성을 쌓고 싶다는 확신을 갖게 됐습니다.

---

## 2. 본인이 주도적으로 해결한 경험

구형 CHEETAH 시스템을 살리는 과정에서 Ubuntu 18.04와 최신 CUDA 드라이버의 충돌 한계를 직접 확인했습니다. apt 설치, Runfile 수동 설치까지 시도했지만 컨테이너 생태계 연동이 근본적으로 불가능하다는 걸 확인한 뒤, "패치가 아니라 재구축이 맞다"고 판단했습니다. 또한 클러스터 구축 중 NAS에 10G NIC가 없다는 사실을 발견하고, master-02의 유휴 카드를 직접 이전해 GPU 노드와 NAS 간 전송속도를 10배 끌어올렸습니다. 문제를 해결하는 것에 그치지 않고, 구조적으로 더 나은 방향을 찾는 것이 제 방식입니다.

---

## 3. 본인의 강점

저는 이상한 것을 그냥 넘기지 않습니다. NAS가 10G 스위치에 연결되어 있는데 실제로는 1G로 통신하고 있다는 모순을 발견했을 때, "왜 이렇게 구성되어 있을까?"라는 질문을 던지고 원인을 추적했습니다. 5년간 방치된 클러스터를 처음 마주했을 때도 마찬가지였습니다. 아무도 손대지 않았던 이유가 어렵기 때문이 아니라 시도하지 않았기 때문이라는 걸 알았고, 직접 시작했습니다. 모르는 것은 부끄럽지 않지만, 알려고 하지 않는 것은 부끄럽다고 생각합니다.

---

## 4. 입사 후 포부

현재 구축한 27GPU 클러스터를 기반으로 드론 영상 기반 Object Detection 모델을 직접 훈련시키고, 인프라 구축부터 모델 서빙까지 MLOps 전 과정을 경험한 뒤 합류하겠습니다. 입사 후에는 인프라와 모델을 모두 이해하는 엔지니어로서, 실제 서비스 환경에서 발생하는 문제를 빠르게 진단하고 해결하는 역할을 하고 싶습니다. 처음부터 잘하는 사람보다, 문제를 발견하고 끝까지 해결하는 사람이 되겠습니다.
